Redis 基础用法

概述 Redis 是常见的内存数据库，通常用来做缓存、计数、排行榜、分布式锁等高频读写场景。 这篇文章聚焦最基础也最常用的内容，包括连接 Redis、常见数据类型、基本读写操作、过期时间，以及实战里常见的使用坑点。 连接 Redis 最常见的方式是使用命令行客户端： redis-cli 如果要连接指定主机和端口： redis-cli -h 127.0.0.1 -p 6379 如果服务设置了密码： redis-cli -h 127.0.0.1 -p 6379 -a your_password 连接成功后，可以先做一个简单检查： PING 返回 PONG 说明 Redis 服务可用。 基本概念 Redis 的数据以 key -> value 的形式存储。和关系型数据库不同，它没有表结构，重点是根据业务场景选择合适的数据类型。 开发里最常见的几类数据： String：最常用，适合缓存字符串、数字、JSON Hash：适合存对象结构 List：适合消息队列、按顺序保存数据 Set：适合去重 ZSet：适合排行榜 String 常用操作 String 是 Redis 使用最频繁的数据类型。 1. 写入和读取 SET user:1001:name "alice" GET user:1001:name 2. 覆盖写入 SET user:1001:name "bob" SET 会直接覆盖旧值，这一点要特别注意。 3. 数字自增 SET page:view 100 INCR page:view INCRBY page:view 10 这类操作适合做访问量、点赞数、库存计数等场景。 4. 删除 key DEL user:1001:name Hash 常用操作 Hash 适合存一组相关字段，比如用户信息。 ...
