TypeScript 是在 JavaScript 基础上增加了类型能力的语言。对初学者来说，先不用纠结太多概念，先把最常见的基础写法掌握住就够了。

一、基础类型#

TypeScript 可以直接给变量标注类型：

const name: string = "Tom";
const age: number = 18;
const passed: boolean = true;

常见基础类型有：

  • string：字符串
  • number：数字
  • boolean：布尔值
  • undefined
  • null

如果类型和赋值不一致，TypeScript 会直接报错。

const age: number = "18"; // 报错

数组类型#

数组有两种常见写法：

const ids: number[] = [1, 2, 3];
const names: Array<string> = ["Tom", "Jerry"];

这两种写法都表示“数组中元素的类型是固定的”。

例如：

const scores: number[] = [95, 88, 76];
const cities: string[] = ["Shanghai", "Beijing", "Shenzhen"];

如果往 scores 里放字符串，TypeScript 会报错：

const scores: number[] = [95, 88, "76"]; // 报错

string 类型#

string 表示字符串类型，也就是文本内容。

const username: string = "Tom";
const message: string = "Hello TypeScript";

一般只要变量里存的是一段文本，就可以考虑使用 string。

例如下面这些值都属于字符串：

const city: string = "Shanghai";
const phone: string = "13800138000";
const id: string = "user_1001";

注意：即使内容看起来像数字，只要你希望它按文本处理，也应该使用 string，例如手机号、订单号、身份证号。

理解 undefined 与 null#

undefined 和 null 都表示“没有值”，但它们的使用场景不完全一样。

  • undefined：通常表示“还没有值”或“这个字段不存在”
  • null：通常表示“这里明确是空值”

例如：

let nickname: string | undefined = undefined;
let avatar: string | null = null;

可以先这样理解：

  • undefined 更像“没提供”
  • null 更像“有这个位置，但现在为空”

在 TypeScript 里，它们通常会和联合类型一起使用：

let name: string | undefined = undefined;
let email: string | null = null;

如果后面要访问这些变量，通常需要先判断它们是不是空值。

二、变量说明#

TypeScript 沿用了 JavaScript 的变量声明方式，所以变量不只有 const 一种写法。

  • const：声明后不能再次赋值
  • let：后续可以重新赋值
  • var：旧写法，现在通常不推荐

例如：

const siteName: string = "TypeScript Guide";
let count: number = 1;

count = 2; // 可以
// siteName = "New Name"; // 报错

实际开发里可以这样理解：

  • 值不会变时，优先用 const
  • 值会变化时，用 let

所以文章里很多示例使用 const，只是因为这些例子里的值本来就不需要修改，不是说 TypeScript 里所有变量都应该用 const。

三、函数与对象#

函数参数和返回值#

函数是 TypeScript 里最常见的使用场景。

function add(a: number, b: number): number {
  return a + b;
}

这里约束了：

  • a 和 b 必须是 number
  • 返回值也必须是 number

如果函数没有返回值，可以写成：

function logMessage(message: string): void {
  console.log(message);
}

再看一个实际一点的例子：

function getUserMessage(name: string, age: number): string {
  return `${name} is ${age} years old`;
}

const message = getUserMessage("Tom", 18);

如果少传参数，或者参数类型不对，也会直接提示。

对象类型#

对象可以直接写类型：

const user: { name: string; age: number } = {
  name: "Tom",
  age: 18
};

但如果对象结构会重复使用，更推荐 interface。

例如：

const book: { title: string; price: number } = {
  title: "TypeScript Guide",
  price: 59
};

interface 基础用法#

interface 可以理解成“对象结构定义”。

interface User {
  name: string;
  age: number;
}

const user: User = {
  name: "Tom",
  age: 18
}

如果某个字段不是必填，可以加 ?：

interface User {
  name: string;
  age: number;
  phone?: string;
}

这表示 phone 可以有，也可以没有。

再看一个更常见的接口例子：

interface Article {
  id: number;
  title: string;
  published: boolean;
}

const article: Article = {
  id: 1,
  title: "TypeScript基础用法",
  published: true
};

type 基础用法#

除了 interface，也可以用 type 定义类型：

type User = {
  name: string;
  age: number;
};

基础场景下，type 和 interface 都能定义对象类型。初学阶段先会用就够了，不必过早纠结两者差异。

例如也可以这样定义接口返回结果：

type ApiResponse = {
  code: number;
  message: string;
};

const res: ApiResponse = {
  code: 200,
  message: "success"
};

四、常见类型能力#

有时候一个值可能不止一种类型，这时可以用联合类型：

let id: string | number;

id = "1001";
id = 1001;

表示 id 可以是字符串，也可以是数字。

这种写法在接口字段里也很常见：

interface Result {
  data: string | null;
}

有些值只允许固定几种取值：

let direction: "left" | "right";

direction = "left";
direction = "right";
// direction = "top"; // 报错

这类写法适合限制状态值、方向值、枚举值。

例如按钮大小只能是下面三种之一：

let buttonSize: "small" | "medium" | "large";

buttonSize = "small";
buttonSize = "large";

可选参数：

function greet(name: string, title?: string) {
  if (title) {
    return `${title} ${name}`;
  }
  return name;
}

默认参数：

function greet(name: string, title: string = "Mr.") {
  return `${title} ${name}`;
}

实际调用效果：

greet("Tom");          // Mr. Tom
greet("Tom", "Dr.");   // Dr. Tom

泛型可以先简单理解成：让类型也能作为参数传进去。

function getFirst<T>(list: T[]): T | undefined {
  return list[0];
}

调用时：

const n = getFirst<number>([1, 2, 3]);
const s = getFirst<string>(["a", "b", "c"]);

初学阶段只需要知道：泛型是为了让函数适配多种类型，同时保留类型信息。

再看一个数组包装示例：

function wrap<T>(value: T): T[] {
  return [value];
}

const n = wrap(100);       // number[]
const s = wrap("hello");   // string[]

很多时候，TypeScript 不写类型也能推断出来：

const name = "Tom"; // 推断为 string
const age = 18;     // 推断为 number

所以不是所有地方都要手动写类型。 一般在这几种场景更适合显式标注：

  • 函数参数
  • 函数返回值
  • 结构复杂的对象
  • 公共接口类型

下面这两种写法都可以：

const username = "Alice";
let level: number = 3;

前者依赖类型推断，后者是显式标注。

有时你比 TypeScript 更清楚一个值的类型，这时可以用断言：

const value: unknown = "hello";
const len = (value as string).length;

但断言不是类型检查的替代品，不能乱用。

例如从页面元素中取值时，这类写法比较常见：

const input = document.getElementById("username") as HTMLInputElement;
console.log(input.value);

五、可空与安全访问#

TypeScript 支持声明一个值可能为空，也支持在访问属性时做安全判断。

可空类型#

如果一个值可能没有内容，可以显式写出来：

let name: string | null = null;
let age: number | undefined = undefined;

这表示变量不一定总是有正常值。

可选属性#

对象里的字段也可以是可选的：

interface Profile {
  nickname?: string;
}

这表示 nickname 可以存在，也可以不存在。

可选链#

访问可能为空的对象时，可以使用 ?.：

type User = {
  profile?: {
    name?: string;
  };
};

const user: User = {};
const username = user?.profile?.name;

如果某一层是 undefined 或 null，表达式会直接返回 undefined，不会继续报错。

空值兜底#

如果希望为空时给默认值，可以用 ??：

const username = user?.profile?.name ?? "匿名用户";

只有左边是 null 或 undefined 时，才会使用右边的默认值。

六、异常处理#

TypeScript 的异常处理方式和 JavaScript 一样，核心还是 try、catch、finally。

try / catch#

当一段代码可能抛出异常时，可以这样处理：

try {
  const result = JSON.parse('{"name":"Tom"}');
  console.log(result.name);
} catch (error) {
  console.log("解析失败");
}

如果没有异常，代码会正常执行；一旦出现异常，就会进入 catch。

手动抛出异常#

除了接收异常，也可以自己抛出异常：

function divide(a: number, b: number): number {
  if (b === 0) {
    throw new Error("除数不能为 0");
  }
  return a / b;
}

这类写法适合在参数非法、状态异常时主动终止执行。

catch 中的 error#

在 TypeScript 里，catch 里的 error 不一定默认就是 Error 对象，因此更稳妥的写法是先判断：

try {
  throw new Error("请求失败");
} catch (error) {
  if (error instanceof Error) {
    console.log(error.message);
  }
}

这样可以避免直接访问不存在的属性。

finally#

如果有无论成功还是失败都要执行的逻辑，可以放在 finally：

try {
  console.log("开始执行");
} catch (error) {
  console.log("执行失败");
} finally {
  console.log("结束清理");
}

常见场景包括关闭资源、结束 loading、收尾日志等。

七、小结#

TypeScript 的基础用法，核心就是两件事：

  • 给数据加上明确的类型
  • 让函数、对象、数组这些常见结构更清晰

先把最基础的类型标注写熟，再继续学习更复杂的类型系统，会更轻松。
