HTTP代理设置#

1. 设置PC端代理端口号#

在Proxy -> Proxy settings可设置HTTP Proxy的端口号。

2. 设置手机端代理端口号#

在应用内的Help -> Local IP Address上查看本机的IP地址（en0等标识）

3. Android手机代理设置#

设置-WLAN，进入已连接WiFi手动设置（端口需与PC端设置端口号一致）

编辑网络，选择手动代理，输入PC端的IP地址 + Charles配置的端口号。

当手机进行数据请求时，Charles 将弹出是否允许连接此代理(选择允许 allow)，之后便可进行HTTP抓包。

HTTPS代理设置#

HTTP的代理设置是抓取不了HTTPS，故需要进行HTTPS代理设置。

手机上需要安装Charles SSL证书认证，证书下载地址：http://charlesproxy.com/getssl

建议使用Chrome浏览器，其它浏览器有可能报错。

其它下载地址推荐：https://www.charlesproxy.com/assets/legacy-ssl/charles.crt

1. Charles配置SSL#

选中Proxy -> SSL Proxying Setting选项，开启SSL代理，并添加或移除信任地址（默认端口号443）。

如果拦截所有，则配置主机*，端口号 443

2. Android手机安装证书#

在手机浏览器输入上面的证书下载地址，证书下载成功后，进入设置-安全-加密与凭据-安装证书-CA证书，选择已下载的证书进行安装。

不同安卓手机设置里面选择不同，可自行查找系统安全进入证书安装

3. Android系统证书配置#

上面的方式配置出来的证书是用户类型，如果需要抓取应用release类型的网络日志，则需要配置成系统证书。

具体步骤如下：

  1. 选中Help -> SSL Proxying -> Save Charles Root Certificate...选项
  2. 保存charles-ssl-proxying-certificate.pem到指定目录
  3. 执行openssl x509 -inform PEM -subject_hash_old -in [pem文件路径]，这里最上方一行输出的是“91b45736”
  4. 重命名pem文件，系统的证书名称格式一般为：[pem的hash值].0名称。故笔者这里需要将pem文件命名为91b45736.0
  5. 将重命名后的文件push到如下路径：/system/etc/security/cacerts/
  6. 重启手机，在设置应用上找到信任的凭据,查看系统的CA证书列表，确认是否有Charles Proxy CA存在

附上cer转换成pem的命令：openssl x509 -inform DER -in [cer文件] -out [pem文件]

Charles的使用#

浏览视图可分为：Structure 和 Sequence

常用的操作有：

  • 专注视图：在侧边栏对某个地址进行右键 -> Focus操作，这样会按当前host及其它host进行分类，从而更专注于当前的host类别请求内容。
  • 重复请求：在侧边栏对某个地址进行右键 -> Repeat操作，可重复选中的请求。另外Repeat Advanced可配置多次重复请求，相当于简单的压测。
  • 重定向：打开Tools -> Map Remote页面，开启重定向，添加需要映射的host地址，这样在不改变包体的情况下完成host切换。
  • 网速模拟：打开Proxy -> Throttle Settings页面，进行网速模拟配置。再通过Proxy -> start(stop) Throttling开启或关闭网速模拟环境。
  • 修改请求：在侧边栏对某个地址进行右键 -> Compose操作，可重新组装参数并请求。
