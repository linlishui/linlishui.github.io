Mac环境下iTerm2使用指南

概述#

iTerm2 是 macOS 上常用的终端工具。相比系统自带 Terminal，它在分屏、标签页、搜索、复制粘贴、Profile 配置和会话体验上更适合长期开发使用。

日常使用时，可以把终端工作流拆成三层：

  • iTerm2：负责窗口、标签页、分屏、快捷键、复制、搜索等界面能力
  • Shell：负责命令解释，例如 zsh、bash
  • tmux：负责终端会话复用，让远程任务和本地窗口解耦

结论很简单：本地临时任务优先用 iTerm2，远程和长期任务优先用 tmux。iTerm2 管入口和操作体验，tmux 管现场和恢复能力。

基础使用#

1. 标签页和窗口#

标签页适合按项目或任务隔离上下文，不建议在同一个窗口里反复 cd 来回切换。

快捷键 作用
Command + T 新建标签页
Command + W 关闭当前标签页或窗口
Command + N 新建窗口
Command + 数字 切换到指定标签页
Command + Shift + [ 切换到上一个标签页
Command + Shift + ] 切换到下一个标签页
Command + Enter 全屏切换

常见标签页组织方式：

  • api：后端服务
  • web：前端工程
  • log：日志观察
  • db：数据库命令
  • scratch：临时命令

2. 分屏#

iTerm2 分屏适合短时间同时观察多个命令输出，例如一边运行服务，一边查看日志。

快捷键 作用
Command + D 垂直分屏
Command + Shift + D 水平分屏
Command + [ 切换到上一个分屏
Command + ] 切换到下一个分屏

示例：

# 左侧运行服务
npm run dev

# 右侧观察日志
tail -f app.log

# 下方执行临时请求
curl http://localhost:3000/health

分屏适合临时观察，不适合承载大量长期任务。长期任务应该放进 tmux，避免关闭窗口后现场丢失。

3. 复制和粘贴#

iTerm2 支持鼠标选择复制，也支持 macOS 常见快捷键。

快捷键 作用
Command + C 复制选中内容
Command + V 粘贴
Command + K 清空当前屏幕和滚动缓冲区
Ctrl + C 中断当前前台进程

需要注意，终端里的 Ctrl + C 不是复制，而是向当前前台进程发送中断信号。比如停止 ping、tail -f、npm run dev 时才应该使用它。

常用编辑和搜索快捷键#

1. 命令行编辑#

这些快捷键主要来自 Shell 的行编辑能力，在 zsh、bash 中都很常用。

快捷键 作用
Ctrl + A 移动到行首
Ctrl + E 移动到行尾
Ctrl + U 删除光标前所有内容
Ctrl + K 删除光标后所有内容
Ctrl + W 删除光标前一个单词
Ctrl + Y 粘贴刚才删除的内容
Ctrl + L 清屏，类似 clear
Ctrl + C 中断当前命令
Ctrl + D 退出当前 Shell，或结束输入
Ctrl + Z 挂起当前前台任务

常见用法：

# 当前输入了一长串命令，想整行重写
Ctrl + U

# 当前命令前半段正确，后半段想删掉重写
Ctrl + K

# vim 被 Ctrl + Z 挂起后，回到前台
fg

如果希望 Option + 左右方向键 按单词移动，可以在 iTerm2 里配置：

  1. 打开 Settings -> Profiles -> Keys
  2. 选择 Key Mappings
  3. 加载或参考 Natural Text Editing

配置完成后，Option + 左右方向键 可以按单词移动，Option + Delete 可以按单词删除。这个能力对修改长命令非常有用。

2. 历史命令搜索#

历史命令搜索是终端里最高频的效率点。

快捷键或命令 作用
Ctrl + R 反向搜索历史命令
Ctrl + G 退出历史搜索
方向键上/下 上一条或下一条历史命令
history 查看历史命令列表

示例：

# 查找之前执行过的 ssh 命令
Ctrl + R
ssh

# 查看最近历史命令
history | tail -20

如果命令很长，建议先用 Ctrl + R 找出来，再用 Ctrl + A、Ctrl + E、Ctrl + W 做局部修改。

3. 终端内容搜索#

iTerm2 本身支持搜索当前终端的滚动内容。

快捷键 作用
Command + F 搜索当前终端内容
Command + G 跳到下一个搜索结果
Command + Shift + G 跳到上一个搜索结果

这适合查找日志关键字、构建错误、接口返回内容。对于持续输出的日志，建议配合命令行工具过滤：

tail -f app.log | grep "ERROR"

grep -n "NullPointerException" app.log

Command + F 查的是终端缓冲区，grep 查的是文件或命令输出。排查问题时，两者经常配合使用。

进阶用法#

1. 配置 Profiles#

Profile 可以理解为一套终端配置，包括字体、颜色、启动命令、工作目录、环境变量等。

推荐至少准备几类 Profile：

Profile 用途
Default 日常本地开发
Project 固定进入某个项目目录
SSH 连接固定服务器
tmux 启动后自动进入 tmux

如果经常进入同一个项目目录，可以给 Profile 设置启动命令：

cd ~/workspace/demo && exec zsh

这里使用 exec zsh，是为了让当前 Shell 替换为新的 zsh，避免退出时多一层 Shell。

对于远程服务器，也可以把 SSH 命令放进 Profile：

ssh user@example.com

如果服务器上固定使用 tmux，可以进一步写成：

ssh user@example.com -t 'tmux attach -t dev || tmux new -s dev'

这样打开 Profile 后会优先进入已有 dev 会话；如果不存在，就创建一个新的会话。

2. Hotkey Window#

Hotkey Window 可以把 iTerm2 做成随叫随到的下拉终端，适合临时执行命令、查端口、看日志、跑脚本。

常见场景：

# 查看端口占用
lsof -i :8080

# 查看当前 IP
ifconfig | grep inet

# 临时进入项目目录
cd ~/workspace/demo

建议 Hotkey Window 只放临时命令，不要把长期运行的服务放在这里，否则容易在不经意间关闭或覆盖现场。

3. 粘贴安全#

终端粘贴命令时要特别小心，尤其是从网页、聊天工具复制多行命令。

风险点包括：

  • 命令里可能包含不可见字符
  • 多行命令可能自动执行
  • rm、curl | sh、sudo 这类命令需要先看清楚
  • 复制的命令可能依赖当前目录，换个目录执行结果完全不同

更稳妥的方式是先粘贴到临时编辑器里确认，或者先写入脚本文件再执行：

cat <<'EOF' > test.sh
echo "hello"
EOF

bash test.sh

不要直接复制陌生来源的脚本到终端执行，尤其不要直接执行 curl xxx | sh。

tmux配合使用#

1. 为什么还需要 tmux#

iTerm2 的分屏和标签页是图形界面的管理能力；tmux 是终端会话管理能力。

两者区别如下：

能力 iTerm2 tmux
本地分屏 支持 支持
标签页管理 支持 通过 window 管理
关闭窗口后任务保持 不适合 支持
SSH 断线恢复 不支持 支持
跨终端重新接入 不支持 支持

如果在远程服务器上运行任务，建议登录服务器后第一件事就是启动 tmux。

2. 基本命令#

# 新建一个会话
tmux new -s dev

# 查看已有会话
tmux ls

# 重新进入会话
tmux attach -t dev

# 退出但保留会话
Ctrl + b，然后按 d

# 杀掉会话
tmux kill-session -t dev

推荐按项目命名会话：

tmux new -s project_api
tmux new -s project_web
tmux new -s server_log

3. 常用 tmux 快捷键#

tmux 默认前缀键是 Ctrl + b，下面的快捷键都需要先按 Ctrl + b。

快捷键 作用
Ctrl + b 然后 c 新建 window
Ctrl + b 然后 n 下一个 window
Ctrl + b 然后 p 上一个 window
Ctrl + b 然后 % 垂直分屏
Ctrl + b 然后 " 水平分屏
Ctrl + b 然后方向键 切换 pane
Ctrl + b 然后 x 关闭当前 pane
Ctrl + b 然后 d detach，退出但保留会话

一个常见布局：

# window 1: editor
vim

# window 2: server
npm run dev

# window 3: log
tail -f app.log

4. 推荐 tmux 配置#

可以在 ~/.tmux.conf 里放一份最小配置，让 tmux 更适合日常开发：

# 使用 Ctrl+a 作为前缀键，比默认 Ctrl+b 更容易按
unbind C-b
set -g prefix C-a
bind C-a send-prefix

# 从 1 开始编号，和多数人的标签页习惯更一致
set -g base-index 1
setw -g pane-base-index 1

# 提高历史滚动行数
set -g history-limit 10000

# 支持鼠标选择 pane 和滚动
set -g mouse on

# 重新加载配置
bind r source-file ~/.tmux.conf \; display-message "tmux config reloaded"

修改后，在 tmux 里执行：

tmux source-file ~/.tmux.conf

如果已经习惯默认 Ctrl + b，不一定要改成 Ctrl + a。快捷键稳定比追求统一更重要。

5. iTerm2和tmux如何分工#

推荐按下面方式使用：

  • 本地临时任务：用 iTerm2 标签页和分屏
  • 本地长期项目：iTerm2 + tmux，可选
  • 远程服务器：优先 tmux
  • 日志观察：短期用 iTerm2 分屏，长期用 tmux window
  • 多项目切换：iTerm2 管入口，tmux 管现场

远程服务器上的实用流程：

# 打开 iTerm2 后登录服务器
ssh user@server

# 登录服务器后启动 tmux
tmux new -s api

# 运行服务或日志
tail -f /var/log/app.log

# 临时离开，保持任务运行
Ctrl + b，然后按 d

# 下次重新接入
tmux attach -t api

这样即使 SSH 断开，tmux 会话里的任务也不会跟着退出。

常见坑点#

1. 混淆 Command 和 Ctrl#

macOS 里 Command + C 是复制，但终端里的 Ctrl + C 是中断进程。不要在运行服务时误按 Ctrl + C，否则服务会被停止。

2. 清屏不等于清空滚动历史#

Ctrl + L 只是清理当前可视区域，滚动历史通常还在；Command + K 会清空 iTerm2 的滚动缓冲区。排查日志时不要误清掉需要回看的输出。

3. iTerm2分屏不能替代tmux会话#

iTerm2 分屏关闭后，里面的前台任务通常也会结束；tmux detach 后任务仍然在会话里运行。远程服务器上运行构建、部署、日志观察时，不要只依赖 iTerm2 分屏。

4. Option方向键可能不可用#

如果 Option + 左右方向键 不能按单词移动，需要检查 iTerm2 的 Profile 按键配置。可以选择内置的自然文本编辑预设，或者手动绑定转义序列。

5. tmux里复制行为和iTerm2不完全一致#

tmux 开启鼠标后，滚动和选择行为会被 tmux 接管。需要复制大段内容时，可以按住 Option 再用鼠标选择，或者进入 tmux copy mode。这个问题不是 iTerm2 失效，而是 tmux 在管理终端输入。

小结#

iTerm2 的价值不只是替代系统 Terminal，而是让终端工作流更稳定：用标签页组织任务，用分屏观察上下文，用搜索定位输出，用 Profile 固化常用环境。

tmux 的价值在于保留现场，特别适合远程服务器和长期任务。一个比较稳的组合是：iTerm2 负责本地入口和操作体验，tmux 负责远程会话和任务恢复。把两者分工想清楚后，终端使用会比单纯记快捷键更可靠。
