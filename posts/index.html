Android音频路由入门：设备、模式、焦点与音量

Android 音频输入输出的核心不是“播放一个声音”或“录一段声音”这么简单，而是系统需要根据场景选择合适的输入设备、输出设备、音频模式、音频焦点和音量类型。 例如音乐播放通常走扬声器、耳机或蓝牙 A2DP；语音通话通常走听筒、免提、蓝牙通话链路或 USB 耳机；录音则需要选择内置麦克风、耳机麦克风或 USB 麦克风。理解这些概念后，再看 AudioTrack、AudioRecord、MediaPlayer、MediaRecorder 才不会混乱。 基本概念 1. 输入和输出 在 Android 音频系统中，设备通常可以分成两类： 输入设备：给系统提供音频数据，例如内置麦克风、蓝牙耳机麦克风、USB 麦克风。 输出设备：接收系统播放的音频数据，例如扬声器、听筒、有线耳机、蓝牙耳机、HDMI。 AudioDeviceInfo 用 isSource() 和 isSink() 表示设备方向： isSource() == true：音频输入设备。 isSink() == true：音频输出设备。 一个物理设备可能同时包含输入和输出能力。例如带麦克风的有线耳机，耳机部分是输出，麦克风部分是输入。 2. 常用 API 常见入口是 AudioManager： AudioManager audioManager = getSystemService(AudioManager.class); // 查询当前连接的输出设备 AudioDeviceInfo[] outputs = audioManager.getDevices(AudioManager.GET_DEVICES_OUTPUTS); // 查询当前连接的输入设备 AudioDeviceInfo[] inputs = audioManager.getDevices(AudioManager.GET_DEVICES_INPUTS); getDevices() 返回的是当前系统已连接、符合条件的设备列表，不代表设备一定适合当前业务场景。比如蓝牙耳机可能支持媒体播放，但通话场景还要看是否可作为通信设备使用。 音频设备类型 1. 常见输出设备 类型 说明 典型场景 TYPE_BUILTIN_SPEAKER 内置扬声器 外放音乐、免提 TYPE_BUILTIN_EARPIECE 听筒 电话、语音通话 TYPE_WIRED_HEADPHONES 不带麦克风的有线耳机 媒体播放 TYPE_WIRED_HEADSET 带麦克风的有线耳机 通话、录音、播放 TYPE_BLUETOOTH_A2DP 蓝牙媒体音频 音乐、视频 TYPE_BLUETOOTH_SCO 蓝牙通话音频 电话、VoIP TYPE_BLE_HEADSET BLE 音频耳机 新版蓝牙音频播放或通话 TYPE_BLE_SPEAKER BLE 音频音箱 外放播放 TYPE_USB_DEVICE USB 音频设备 外接声卡、USB DAC TYPE_USB_HEADSET USB 耳机 通话、播放、录音 TYPE_HDMI / TYPE_HDMI_ARC / TYPE_HDMI_EARC HDMI 音频输出 电视、功放、家庭影院 2. 常见输入设备 类型 说明 典型场景 TYPE_BUILTIN_MIC 内置麦克风 录音、语音识别、通话 TYPE_WIRED_HEADSET 有线耳机麦克风 通话、录音 TYPE_BLUETOOTH_SCO 蓝牙通话麦克风 电话、VoIP TYPE_USB_DEVICE / TYPE_USB_HEADSET USB 输入设备 外接麦克风、USB 耳机 TYPE_TELEPHONY 电话链路相关设备 系统电话场景 TYPE_REMOTE_SUBMIX 远端或虚拟混音设备 投屏、录屏、系统级混音 设备类型只是描述“这是什么设备”。实际能不能被当前应用使用，还取决于 Android 版本、权限、当前音频模式、设备连接状态和厂商实现。 ...
