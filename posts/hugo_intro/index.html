安装Hugo#

不同系统环境安装命令：

# Ubuntu 环境
## apt方式。下载版本可能会旧一些
sudo apt update
sudo apt install hugo
## deb包方式
uname -i  # 64位：x86_64；32位：i686
sudo dpkg -i hugo*.deb


# Mac 环境
brew install hugo

查看HUGO版本确认安装成功：

hugo version

快速开始#

可参考文档：Hugo快速入门

  1. hugo 创建站点：
# 说明：my-site 是站点的名称
hugo new site my-site

创建完成后，站点目录 包含以下文件：

.
├── archetypes/
│   └── default.md    <-- default.md是默认的博文模版
├── assets/           <-- 存放被 Hugo Pipes 处理的文件
├── content/          <-- 存放markdown文件作为博文内容
├── data/             <-- 存放 Hugo 处理的数据
├── i18n/
├── layouts/     
├── static/           <-- 存放静态文件,例如 图片 CSS JS文件
├── themes/           <-- 存放不同的主题
└── config.yaml         <-- site configuration
  1. 创建文章
hugo new content/posts/hugo_intro.md

# 指定模板创建内容
hugo new --kind <template> <name>
  1. 设置主题（子模块形式）
git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
git submodule update --init --recursive # needed when you reclone your repo (submodules may not get cloned automatically)

# 重新clone，需要初始化子模块，等同于上面的复合命令
git submodule init
git submodule update

这里使用的主题为：PaperMod

  1. 本地运行测试
# 支持草稿态展示，默认端口号 1313
hugo server -D
  1. 正式发布

当进入到正式发布时，在调整确认config.yaml / config.toml 文件下的 baseURL 字段值后。 执行以下命令即可正式发布：

hugo

使用入门#

当前配置的目录菜单列表如下：

menu:
  main:
    - name: 首页
      url: /
      weight: 1

    - name: 文章
      url: /posts/
      weight: 2

    - name: 分类
      url: /categories/
      weight: 3

    - name: 标签
      url: /tags/
      weight: 4

    - name: 归档
      url: /archives/
      weight: 5

    - name: 关于
      url: /about/
      weight: 6

1. 分类与标签#

在 config.yaml 文件里打开分类与标签功能：

taxonomies:
  tag: tags
  category: categories

在archetypes/default.md默认博文模版里新增属性定义：

categories:
  - "未分类"
tags:
  - "未知"

2. 归档#

在 content 目录里创建 archive.md文件 此时的目录树如下：

.
├── config.yaml
├── content/
│   ├── archives.md   <--- Create archive.md here
│   └── posts/
├── static/
└── themes/
    └── PaperMod/

接着将以下内容添加到archive.md文件里：

---
title: "Archive"
layout: "archives"
url: "/archives/"
summary: archives
---

3. 静态资源#

Hugo 构建时会自动将 static 目录内容复制到 public 根目录。

以引用图片为例，在 static 目录下新建子目录（如 images），使用绝对路径直接引用，路径需以 / 开头：

引用方式：![图片描述](/images/demo.jpg)
部署后的访问地址：https://your-domain.com/images/demo.jpg

4. Mermaid图表#

Hugo 目前没有提供 Mermaid 图表的默认模板。可通过以下方法主动配置：

创建 layouts/_default/_markup/render-codeblock-mermaid.html文件，在该文件添加以下内容：

<pre class="mermaid">
  {{- .Inner | safeHTML }}
</pre>
{{ .Page.Store.Set "hasMermaid" true }}

部署#

使用 阿里云服务器配合nginx实现Hugo部署。

  1. 确认 Hugo 的baseURL配置：
# 本机IP动态变化，使用相对路径，不指定host
baseURL: /
  1. 在/etc/nginx/conf.d/目录里新建 hugo-server.conf文件，内容如下：
server {

    listen 9099;
    server_name 127.0.0.1;

    # Hugo 生成的静态网站目录
    root /home/user/hugo-site/public;
    
    index index.html index.htm;

    # 日志配置
    access_log /var/log/nginx/hugo_access.log;
    error_log /var/log/nginx/hugo_error.log;

    location / {
        try_files $uri $uri/ =404;
    }
}

在nginx按需配置访问的端口号与静态网站生成目录。

注意，添加完成后执行 sudo nginx -s reload 命令刷新配置。

  1. 执行 hugo 进行静态网站打包。完成后外站可使用目标主机地址进行访问
