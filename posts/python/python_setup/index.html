这篇文章主要整理 Python 环境相关的常用概念和配置方式，包括 python 与 python3、pip 与 pip3、虚拟环境 venv、以及安装独立工具时常见的 pipx。

Python、python3、pip 与 pip3#

在日常使用里，这几个命令很容易混淆：

  • python：启动 Python 解释器
  • python3：明确使用 Python 3 解释器
  • pip：Python 包管理工具
  • pip3：通常表示给 Python 3 安装包

更实用的理解方式是：python3 和 pip3 往往是一组，尽量配套使用。

例如：

# 查看 Python 3 版本
python3 --version
# 查看 pip3 对应的版本与安装位置
pip3 --version

如果机器上同时存在多个 Python 版本，优先使用 python3 和 pip3，歧义更少。

1. 怎么确认当前命令对应的是哪个环境#

环境问题里最常见的坑，不是命令不会用，而是命令实际指向的解释器和包安装位置不一致。

常见检查方式：

# 查看 python3 的实际路径
which python3

# 查看 pip3 的实际路径
which pip3

# 查看 pip3 安装到了哪个 Python 环境
pip3 --version

如果 python3 和 pip3 指向的环境不一致，安装完包后就可能出现“命令执行了，但代码里还是导入失败”的问题。

pip3 与 pipx#

在 Python 中，pip 是 Python Package Installer（Python 包安装器）的缩写，是 Python 生态中最重要的工具之一，用于安装、升级、卸载和管理第三方库（包）。

本文后续命令统一使用 pip3。

1. pip3#

常用命令：

  • pip3 list：查看已安装的包

  • pip3 show 包名：查看包详情

  • pip3 freeze > requirements.txt：导出当前环境的所有包

  • pip3 install 包名：安装第三方库

    pip3 install requests   # 安装 requests 库
pip3 install numpy==1.24.0  # 安装指定版本
pip3 install -r requirements.txt  # 批量安装指定包

  • pip3 install --upgrade 包名：升级包

    pip3 install --upgrade pip  # 升级 pip 自身
pip3 install --upgrade requests

  • pip3 uninstall 包名：卸载包

更稳妥的习惯是：只在当前虚拟环境里执行 pip3 install，尽量不要直接往全局环境里装项目依赖。

在高版本python3上，使用pip3安装包会报错 “externally-managed-environment” ，具体信息可查阅pep-0668。

此时更推荐使用虚拟环境安装包。

虚拟环境#

虚拟环境用于隔离项目依赖，避免不同项目之间的包版本互相影响。日常开发里，只要项目需要安装第三方包，通常就应该先创建虚拟环境。

最常见的做法是使用 venv：

# 创建虚拟环境
python3 -m venv .venv

# 激活虚拟环境
source .venv/bin/activate  # Linux/macOS

# 在虚拟环境中安装包
pip3 install 包名

使用完毕后，可通过 deactivate 退出虚拟环境。

如果只是临时运行一个独立项目，venv 通常已经够用了。

常见的日常流程通常是：

# 1. 创建虚拟环境
python3 -m venv .venv

# 2. 激活虚拟环境
source .venv/bin/activate

# 3. 安装依赖
pip3 install requests

# 4. 导出依赖
pip3 freeze > requirements.txt

# 5. 退出虚拟环境
deactivate

如果只是运行已有项目，通常还会先执行：

pip3 install -r requirements.txt

激活后，通常可以通过下面两种方式确认当前已经进入虚拟环境：

# 查看当前 python3 是否指向 .venv
which python3

# 查看当前 pip3 是否指向 .venv
which pip3

很多终端提示符前面也会出现类似 (.venv) 的前缀。

1. venv 与 pip3 的关系#

可以把 venv 理解成“创建隔离环境”，把 pip3 理解成“在当前环境里安装包”。

也就是说：

  • venv 负责隔离环境
  • pip3 负责往这个环境里装依赖

两者通常是配套使用的。

2. pipx#

pipx 适合安装独立的 Python 命令行工具，它会为每个工具创建单独的虚拟环境，避免把工具直接装到全局环境里。

常见场景包括：

  • 安装格式化工具，例如 black
  • 安装命令行工具，例如 httpie
  • 安装项目管理工具，例如 poetry

可以简单这样区分：

  • 项目依赖，用 venv + pip3
  • 独立命令行工具，用 pipx

下面以 macOS 为例进行配置：

# 安装pipx，会触发一系列依赖安装
brew install pipx

# 将其添加到 PATH 中
pipx ensurepath

安装工具的例子：

# 安装代码格式化工具
pipx install black
# 安装命令行 HTTP 工具
pipx install httpie
