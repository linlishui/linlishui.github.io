Python 是常用的动态类型脚本语言，这篇文章面向已经用过 Python、但想快速回顾常用基础写法的读者。

变量与基础类型#

Python 变量不需要先声明类型，直接赋值即可。

name = "Tom"
age = 18
price = 19.9
passed = True

常见基础类型有：

  • str：字符串
  • int：整数
  • float：小数
  • bool：布尔值
  • NoneType：空值类型，对应 None

常见基础数据结构有：

  • list：列表，有顺序、可修改
  • tuple：元组，有顺序、不可修改
  • set：集合，无序、不重复
  • dict：字典，键值对结构

如果想看变量类型，可以直接用 type()：

name = "Tom"
age = 18

# 查看变量运行时的实际类型
print(type(name))
print(type(age))

实际开发里，大多数时候不用刻意写类型判断，但在调试阶段它很有用。

1. Python 是动态类型语言#

Python 是动态类型语言。

变量本身不固定类型，类型跟当前保存的值有关。

value = 10
# 当前是 int
print(type(value))

value = "hello"
# 同一个变量现在变成了 str
print(type(value))

同一个变量先保存整数、后保存字符串，在 Python 里是允许的。

2. 变量可以是空值，也可能未定义#

变量未定义时，直接访问会报错：

print(name)  # NameError

变量已经定义但当前没有值时，一般用 None 表示：

name = None
print(name)

nickname = None

# 用 is None 判断空值
if nickname is None:
    print("nickname is empty")

None 就表示“现在没有值”。

3. 字符串#

字符串就是文本内容。

username = "alice"
city = "Shanghai"
message = "hello python"

最常见的字符串拼接方式是 f-string：

name = "Tom"
age = 18

# f-string 会直接把变量插入字符串
print(f"{name} is {age} years old")

相比 + 拼接，f-string 更直观，也更适合实际项目。

4. 数字与布尔值#

数字可以直接做计算，布尔值常用于条件判断。

count = 3
price = 9.9
is_admin = False

print(count + 2)
print(price * 3)
print(is_admin)

5. list、tuple、set、dict#

除了字符串、数字和布尔值，Python 里最常用的就是这几类数据结构。

  • list：适合保存一组有顺序的数据
  • tuple：和 list 类似，但创建后不能修改
  • set：适合去重，和 Java 里的 Set 很接近
  • dict：适合保存字段名和字段值的对应关系

例如：

names = ["Tom", "Jerry"]
point = (10, 20)
tags = {"python", "java", "python"}
user = {"name": "Tom", "age": 18}

# list
print(names)
# tuple
print(point)
# set 会自动去重
print(tags)
# dict
print(user)

这里的 set 会自动去重：

tags = {"python", "java", "python"}
print(tags)

如果你有 Java 背景，可以先这样理解：

  • Python 的 list 有点像 Java 里的 ArrayList
  • Python 的 dict 有点像 Java 里的 Map
  • Python 的 set 有点像 Java 里的 Set

最常见的操作通常是这些：

names = ["Tom", "Jerry"]
# 追加元素
names.append("Alice")
# 按索引访问
print(names[0])

point = (10, 20)
# tuple 可以访问，但不能原地修改
print(point[0])

tags = {"python", "java"}
# set 常用于去重或快速判断是否存在
tags.add("go")
print(tags)

user = {"name": "Tom"}
# 新增或更新字段
user["age"] = 18
# 字段不存在时返回 None
print(user.get("name"))

列表与字典#

入门阶段最常用的数据结构就是列表和字典。

1. 列表#

列表可以理解成“有顺序的一组数据”。

names = ["Tom", "Jerry", "Alice"]

# 按下标访问元素
print(names[0])
print(names[1])

常见操作：

names = ["Tom", "Jerry"]

# 末尾追加元素
names.append("Alice")
print(names)

# 获取列表长度
print(len(names))

遍历列表也很常见：

names = ["Tom", "Jerry", "Alice"]

# 逐个遍历列表元素
for name in names:
    print(name)

处理一批用户、一批文件名、一批订单时，列表往往是最先使用的数据结构。

2. 字典#

字典适合保存“字段名 -> 字段值”这样的结构化数据。

user = {
    "name": "Tom",
    "age": 18
}

# 通过 key 直接取值
print(user["name"])
print(user["age"])

也可以用 get() 取值：

user = {
    "name": "Tom",
    "age": 18
}

# 字段存在时返回对应值
print(user.get("name"))
# 字段不存在时返回 None
print(user.get("email"))

get() 的好处是字段不存在时不会直接报错，更适合处理不确定数据。

条件判断与循环#

Python 里的控制流程非常常用，尤其是在写脚本时。

1. if 判断#

score = 85

if score >= 90:
    print("优秀")
elif score >= 60:
    print("及格")
else:
    print("不及格")

这类逻辑在接口状态判断、参数检查、权限控制里都很常见。

2. for 循环#

for i in range(5):
    # range(5) 会生成 0 到 4
    print(i)

range(5) 表示从 0 到 4。

也经常和列表一起用：

users = ["Tom", "Jerry", "Alice"]

# 遍历 users 里的每个元素
for user in users:
    print(f"current user: {user}")

3. while 循环#

count = 0

while count < 3:
    print(count)
    # 手动更新变量，避免死循环
    count += 1

while 更适合“不确定要循环几次”的场景，但要特别注意退出条件，否则容易写成死循环。

函数基础#

函数就是把重复逻辑封装起来，避免同样的代码写很多次。

def add(a, b):
    # return 把结果返回给调用方
    return a + b

result = add(3, 5)
print(result)

这是最常见的函数写法。

1. 带默认参数的函数#

def greet(name="world"):
    print(f"hello, {name}")

# 不传参数时使用默认值
greet()
# 传参时覆盖默认值
greet("Tom")

默认参数在写工具函数时很常见，可以减少调用时的参数数量。

2. 一个实战场景里的例子#

def format_user(name, age, city):
    return f"name={name}, age={age}, city={city}"

# 调用函数并接收返回值
message = format_user("Tom", 18, "Shanghai")
print(message)

如果这段逻辑在多个地方都要用，就应该封装成函数，而不是每次都重新拼字符串。

文件读写#

Python 很常被用来写脚本，文件操作是很常见的场景。

1. 读取文件#

with open("demo.txt", "r", encoding="utf-8") as f:
    # 读取整个文件内容
    content = f.read()

print(content)

2. 写入文件#

with open("demo.txt", "w", encoding="utf-8") as f:
    # 写入文本内容
    f.write("hello python")

这里的 with open(...) 很重要，它能帮你在使用完成后自动关闭文件。

自动化脚本、日志处理脚本、配置生成脚本里，这类代码都很常见。

异常处理#

写 Python 脚本时，异常处理很常见，因为很多操作都可能失败，比如文件不存在、类型不对、转换失败、网络请求失败。

1. try / except 基础写法#

try:
    num = int("abc")
except ValueError:
    # 捕获类型转换失败
    print("convert failed")

这里 int("abc") 会报错，因为 "abc" 不能转成整数。

实际开发里，异常处理的核心目的不是“让程序永远不报错”，而是“在出错时给出可控结果”。

2. 一个实战场景里的例子#

try:
    with open("data.txt", "r", encoding="utf-8") as f:
        content = f.read()
    print(content)
except FileNotFoundError:
    # 文件不存在时进入这里
    print("file not found")

如果文件不存在，程序不会直接崩掉，而是进入 except 分支。

3. finally#

如果有些逻辑无论是否出错都要执行，可以放到 finally 里。

try:
    print("start")
    result = 10 / 2
except ZeroDivisionError:
    print("division error")
finally:
    # finally 里的代码一定会执行
    print("finished")

finally 常用于资源清理、收尾日志、状态恢复等场景。

一个常见脚本场景#

下面是一个更接近日常使用的数据清洗示例：

users = [
    {"name": "Tom", "age": 18, "city": "Shanghai"},
    {"name": "Jerry", "age": None, "city": "Beijing"},
    {"name": "", "age": 22, "city": "Shenzhen"},
]

def format_user(user):
    name = user.get("name")
    age = user.get("age")
    city = user.get("city")

    # 字段缺失时直接跳过
    if not name or age is None:
        return None

    return f"{name}({age}) - {city}"

result = []

for user in users:
    formatted = format_user(user)
    # 只保留有效数据
    if formatted is not None:
        result.append(formatted)

print(result)

这个例子不复杂，但已经把几种常见能力串起来了：

  • 列表
  • 字典
  • 函数
  • 条件判断
  • 循环
  • None 判断

这种写法比单独记语法更接近日常脚本处理场景。

常见说明#

1. 缩进必须保持一致#

Python 用缩进表示代码块，所以不能随便对齐。

if True:
    print("hello")
    print("world")

缩进一般统一使用 4 个空格，不要一会儿用空格，一会儿用 tab。

2. = 和 == 不是一回事#

  • =：赋值
  • ==：判断是否相等

例如：

age = 18

if age == 18:
    print("yes")

这是初学者非常容易写错的地方。

3. 字典取值要注意字段是否存在#

下面这种写法如果字段不存在会报错：

user = {"name": "Tom"}
print(user["email"])

更稳妥一点的写法是：

user = {"name": "Tom"}
print(user.get("email"))

处理接口返回值、配置文件、外部输入数据时，优先考虑 get()。

4. None 不是空字符串#

下面这几个值不是一回事：

name = None
name = ""
name = "Tom"
  • None：没有值
  • ""：有值，只是空字符串
  • "Tom"：普通字符串

这在参数判断、接口字段判断里很容易混淆。

小结#

把变量、流程控制、函数、常用数据结构和异常处理这些常用基础重新串起来，日常写 Python 脚本时就会更顺手。
