这篇文章适合已经接触过 Python、但想快速把异步概念捡起来的读者。目标很简单：用最少的内容讲清楚 Python 异步是干什么的、怎么写、哪里最容易出错。

异步是为了解决什么问题#

Python 异步主要解决的是 I/O 等待，比如：

  • 同时请求多个接口
  • 同时查多个远程服务
  • Web 服务在等数据库或网络返回时，不想把线程一直卡住

它不擅长解决 CPU 密集型计算问题。比如大量图片处理、复杂计算，这类任务通常不靠异步提速。

可以先记住一句话：

异步的重点不是“更快执行一段代码”，而是“等待的时候别闲着”。

最核心的三个点#

1. async def 定义协程函数#

async def fetch_data():
    return "ok"

这不是普通函数。调用它时，不会立刻执行，而是先得到一个协程对象。

2. await 表示“先等一下，同时让出执行权”#

import asyncio

async def main():
    await asyncio.sleep(1)
    print("done")

asyncio.run(main())

这里的 await asyncio.sleep(1) 不等于“整个程序卡住 1 秒”，而是“当前任务先暂停，事件循环可以先去执行别的任务”。

3. asyncio.run() 是常见入口#

asyncio.run(main())

可以把它理解成：启动事件循环，把协程跑起来。

最常见的写法区别#

很多人刚写异步时，问题不在语法，而在“看起来是异步，其实还是串行”。

串行写法#

import asyncio

async def work(name, delay):
    await asyncio.sleep(delay)
    return f"{name} done"

async def main():
    r1 = await work("task1", 1)
    r2 = await work("task2", 2)
    print(r1, r2)

asyncio.run(main())

这段代码会一个接一个执行，总耗时大约是 3 秒。

并发写法#

import asyncio

async def work(name, delay):
    await asyncio.sleep(delay)
    return f"{name} done"

async def main():
    r1, r2 = await asyncio.gather(
        work("task1", 1),
        work("task2", 2),
    )
    print(r1, r2)

asyncio.run(main())

这里两个任务会一起调度，总耗时大约是 2 秒。

回顾异步时，最值得记住的就是这一点：

  • await a(); await b() 是串行
  • await asyncio.gather(a(), b()) 才是并发

最容易踩的坑#

1. 忘了 await#

async def get_data():
    return 123

async def main():
    result = get_data()
    print(result)

这里拿到的不是结果，而是协程对象。少了 await，代码其实没真正执行。

2. 在异步代码里写阻塞调用#

import time

async def bad_task():
    time.sleep(2)

time.sleep(2) 会直接卡住线程，事件循环也会一起被卡住。异步代码里应该优先用异步版本，比如 asyncio.sleep()。

3. 以为异步等于多线程#

异步不是自动开线程。它更像是单线程里的协作切换：任务在等待 I/O 时主动让出执行权。

一个够用的理解框架#

如果你只是想快速回忆 Python 异步，可以直接记住这几句：

  • 异步主要解决 I/O 等待，不是 CPU 加速
  • async def 定义协程
  • await 表示等待，同时让出执行权
  • 多个独立任务要并发，通常用 asyncio.gather()
  • 协程里不要混入阻塞调用

异步进阶补充#

如果再往前走一步，最常遇到的就是任务取消、超时控制和异常处理。

1. 任务取消#

import asyncio

async def work():
    try:
        await asyncio.sleep(10)
    except asyncio.CancelledError:
        print("task cancelled")
        raise

async def main():
    task = asyncio.create_task(work())
    await asyncio.sleep(1)
    task.cancel()
    await task

asyncio.run(main())

task.cancel() 不是立刻强杀任务，而是向任务发出取消请求。协程通常会在下一次 await 时收到 CancelledError。

2. 超时控制#

import asyncio

async def slow_task():
    await asyncio.sleep(5)

async def main():
    try:
        await asyncio.wait_for(slow_task(), timeout=2)
    except asyncio.TimeoutError:
        print("timeout")

asyncio.run(main())

超时控制适合放在外部 I/O 调用周围，避免某个慢请求一直拖住整个流程。

3. 异常处理#

import asyncio

async def ok():
    await asyncio.sleep(1)
    return "ok"

async def bad():
    await asyncio.sleep(1)
    raise ValueError("request failed")

async def main():
    results = await asyncio.gather(ok(), bad(), return_exceptions=True)
    print(results)

asyncio.run(main())

多个任务一起执行时，如果你不想一个任务报错就让整组任务直接中断，可以用 return_exceptions=True 先把异常收集回来，再统一处理。

小结#

Python 异步并不复杂，真正常用的核心点其实就几个：async def、await、asyncio.run() 和 asyncio.gather()。

如果你现在已经能分清“协程是什么”“什么时候该 await”“怎么把多个 I/O 任务一起跑”，再额外知道怎么取消任务、限制超时、收集异常，那这篇回顾的目标就已经达到了。
