这篇文章主要整理 Python 里常见的模块导入方式、日常脚本开发中经常会用到的标准库，以及常见网络库的基础用法。

模块导入#

Python 代码通常会拆到不同文件中，拆分后的代码会通过导入来组织和复用。

1. 同目录下导入整个模块#

假设有一个 tools.py 文件：

# 定义一个简单函数
def add(a, b):
    return a + b

在另一个文件中这样使用：

# 导入整个 tools 模块
import tools

# 通过 模块名.函数名 调用
print(tools.add(1, 2))

这种写法比较直接，适合区分“函数来自哪个模块”。

2. 同目录下导入指定内容#

也可以只导入需要的函数：

# 只导入 add 函数
from tools import add

# 直接调用函数名
print(add(1, 2))

这种写法更简洁，但项目变大后，如果导入太多同名函数，代码可读性会差一点。

3. 包内导入#

当代码放进目录结构里时，通常会按包来导入。

例如目录结构：

project/
  app/
    main.py
    utils.py

main.py 中可以这样写：

# 从 app 包中的 utils 模块导入 add
from app.utils import add

这种写法更适合项目代码。模块不在当前文件同目录下时，也优先通过包路径导入，而不是手动改 sys.path。只有在临时脚本场景下，才会看到 sys.path 相关写法，但这类方式不适合作为常规项目结构。

4. 相对导入#

在包内部，也会看到相对导入写法。

例如：

# 从当前包中的 utils 模块导入 add
from .utils import add

这种写法常见于包内部模块互相引用，但前提是代码本身运行在包结构里，而不是直接把单个文件当脚本执行。

导入失败的常见原因通常包括：工作目录不对、包路径不正确、直接运行单个文件导致相对导入失效。

常用标准库#

Python 自带很多标准库，日常脚本里常见的有下面几类：

模块 说明
os 获取系统信息、环境变量、当前工作目录
pathlib 处理文件和目录路径
json 处理 JSON 字符串与 Python 对象转换
random 生成随机数和随机选择
datetime 获取时间和格式化时间
subprocess 在脚本里执行系统命令

1. os#

适合获取环境变量、当前目录等系统信息。

常见方法：

  • os.getcwd()：获取当前工作目录，返回 str
  • os.environ.get("HOME")：获取环境变量，返回 str 或 None
import os

# 获取当前工作目录
print(os.getcwd())

# 获取环境变量，不存在时返回 None
print(os.environ.get("HOME"))

2. pathlib#

适合处理文件路径，写法比字符串拼接更稳妥。

常见方法：

  • Path("data")：创建路径对象，返回 Path
  • path.exists()：判断路径是否存在，返回 bool
from pathlib import Path

# 拼接路径对象
path = Path("data") / "users.json"
# 查看路径是否存在
print(path.exists())
# 输出路径字符串表现
print(path)

3. json#

适合处理接口返回值、本地配置文件、导出的结构化数据。

常见方法：

  • json.dumps(obj)：对象转 JSON 字符串，返回 str
  • json.loads(text)：JSON 字符串转 Python 对象，通常返回 dict 或 list
import json

data = {"name": "Tom", "age": 18}
# 对象转 JSON 字符串
text = json.dumps(data, ensure_ascii=False)
# JSON 字符串转回对象
obj = json.loads(text)
print(obj)

日常处理接口返回值时，如果 json.loads() 的结果是 dict，更推荐优先使用 get() 取值：

import json

text = '{"name": "Tom"}'
obj = json.loads(text)

print(obj.get("name"))
print(obj.get("city"))

obj["name"] 在字段不存在时会抛出 KeyError，obj.get("name") 则会返回 None。如果后续代码依赖字段类型，再补充 isinstance() 判断即可。

4. random#

适合生成测试数据或随机结果。

常见方法：

  • random.randint(1, 10)：生成区间内随机整数，返回 int
  • random.choice(seq)：从序列中随机取一个元素，返回对应元素
import random

# 生成 1 到 10 之间的随机整数
print(random.randint(1, 10))

# 从列表里随机取一个值
print(random.choice(["Tom", "Jerry", "Alice"]))

5. datetime#

适合处理时间格式化和时间计算。

常见方法：

  • datetime.now()：获取当前时间，返回 datetime
  • strftime("%Y-%m-%d")：格式化时间，返回 str
from datetime import datetime

# 按指定格式输出当前时间
print(datetime.now().strftime("%Y-%m-%d"))

6. subprocess#

适合在 Python 脚本里执行系统命令。

常见方法：

  • subprocess.run(...)：执行命令，返回 CompletedProcess
  • result.stdout：读取标准输出，返回 str
import subprocess

# 执行命令并获取输出
result = subprocess.run(["pwd"], capture_output=True, text=True)
print(result.stdout)

常见网络库#

网络请求也是模块使用里很常见的一类场景。

除了标准库，日常开发里最常见的网络请求方案通常是 requests 和 urllib。

库 说明
requests 最常见的 HTTP 请求库，适合调用接口
urllib Python 标准库里的网络请求工具，功能基础但够用

1. requests#

requests 是最常见的第三方网络库，写法直接，适合大多数接口调用场景。

常见方法：

  • requests.get(url)：发送 GET 请求
  • requests.post(url, json=data)：发送 POST 请求
  • response.json()：把响应内容转成 Python 对象，通常是 dict 或 list

返回 JSON 时通常使用 response.json()，返回普通文本时则更常用 response.text。

GET 请求示例：

import requests

# 发送 GET 请求，并设置 5 秒超时
response = requests.get("https://example.com/api/users", timeout=5)

# 把响应内容转成 Python 对象
data = response.json()
print(data)

实际使用时，通常还会关注 response.status_code，或者直接调用 response.raise_for_status() 判断请求是否成功。

POST 请求示例：

import requests

payload = {"name": "Tom", "age": 18}

# 发送 POST 请求，并以 JSON 形式传参
response = requests.post(
    "https://example.com/api/users",
    json=payload,
    timeout=5
)
print(response.json())

基础错误处理示例：

import requests

try:
    response = requests.get("https://example.com/api/users", timeout=5)
    response.raise_for_status()
    print(response.json())
except requests.RequestException as e:
    print(f"request failed: {e}")

2. urllib#

urllib 是标准库自带的网络请求工具，不安装第三方包时可以直接使用。

GET 请求示例：

from urllib import request

# 发送请求并设置 5 秒超时
with request.urlopen("https://example.com", timeout=5) as resp:
    text = resp.read().decode("utf-8")
    print(text)

urllib 请求失败时通常会直接抛异常，所以更适合基础场景或不想额外安装第三方库时使用。

POST 请求示例：

from urllib import request
import json

payload = {"name": "Tom", "age": 18}
data = json.dumps(payload).encode("utf-8")

# 构造 POST 请求
req = request.Request(
    "https://example.com/api/users",
    data=data,
    headers={"Content-Type": "application/json"},
    method="POST"
)

with request.urlopen(req, timeout=5) as resp:
    text = resp.read().decode("utf-8")
    print(text)

基础错误处理示例：

from urllib import request, error

try:
    with request.urlopen("https://example.com", timeout=5) as resp:
        print(resp.read().decode("utf-8"))
except error.URLError as e:
    print(f"request failed: {e}")

小结#

把模块导入方式、常用标准库和基础网络请求串起来，写日常 Python 脚本时会更顺手。
