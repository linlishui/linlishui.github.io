Docker 从入门到实践

概述 如果你已经有一定工程经验，Docker 最有价值的点，不是“会几个命令”，而是把项目运行环境、依赖和服务编排变成一套可重复执行的工程描述。 这篇文章不打算把 Docker 的全部概念讲一遍，而是围绕一个完整的本地项目场景，串起从入门到实践最常用的部分： 用 Docker 跑一个 Nginx 静态站点 用 Docker 构建一个 Node.js 或 Python/FastAPI 应用 用 Docker Compose 编排 MySQL + Redis + App 理解镜像、容器、端口、卷、网络这些真正会影响日常开发的核心概念 先说结论： 镜像是“运行模板”，容器是“运行中的实例” docker run 适合单服务验证，docker compose 更适合真实项目 真正落地时，最容易出问题的不是“命令不会写”，而是端口、卷、网络、启动顺序和环境变量 对工程项目来说，Compose 文件和 Dockerfile 本身就是运行文档 Docker 到底解决什么问题 对有基础的工程师来说，Docker 的价值主要有三点： 1. 环境一致性 本地能跑、测试能跑、线上也能跑，依赖的是同一套镜像和配置，而不是“我电脑上正好装过”。 2. 服务隔离 MySQL、Redis、App 不需要都直接装在宿主机上。每个服务跑在自己的容器里，依赖边界更清晰。 3. 低成本复现 拉代码、执行命令、启动编排，整个项目环境就能快速恢复。对多人协作、本地调试、临时排障都很有用。 先理解 4 个最核心概念 1. 镜像 镜像可以理解为容器的只读模板，里面包含应用代码、运行时、依赖和启动命令。 常见操作： docker pull nginx:latest docker images 2. 容器 容器是镜像启动后的运行实例。一个镜像可以启动多个容器。 常见操作： ...

Redis 基础用法

概述 Redis 是常见的内存数据库，通常用来做缓存、计数、排行榜、分布式锁等高频读写场景。 这篇文章聚焦最基础也最常用的内容，包括连接 Redis、常见数据类型、基本读写操作、过期时间，以及实战里常见的使用坑点。 连接 Redis 最常见的方式是使用命令行客户端： redis-cli 如果要连接指定主机和端口： redis-cli -h 127.0.0.1 -p 6379 如果服务设置了密码： redis-cli -h 127.0.0.1 -p 6379 -a your_password 连接成功后，可以先做一个简单检查： PING 返回 PONG 说明 Redis 服务可用。 基本概念 Redis 的数据以 key -> value 的形式存储。和关系型数据库不同，它没有表结构，重点是根据业务场景选择合适的数据类型。 开发里最常见的几类数据： String：最常用，适合缓存字符串、数字、JSON Hash：适合存对象结构 List：适合消息队列、按顺序保存数据 Set：适合去重 ZSet：适合排行榜 String 常用操作 String 是 Redis 使用最频繁的数据类型。 1. 写入和读取 SET user:1001:name "alice" GET user:1001:name 2. 覆盖写入 SET user:1001:name "bob" SET 会直接覆盖旧值，这一点要特别注意。 3. 数字自增 SET page:view 100 INCR page:view INCRBY page:view 10 这类操作适合做访问量、点赞数、库存计数等场景。 4. 删除 key DEL user:1001:name Hash 常用操作 Hash 适合存一组相关字段，比如用户信息。 ...
