Git入门介绍

1. 概述 Git的本地区域可以划分为如下几个部分： 工作目录（Working Directory） 暂存目录（也叫做索引，Index) 本地仓库（Repository） 2. 基本用法 2.0 常用操作 查找某个字串：git grep -nir [word-regexp] 基于当前分支创建新分支，并创建对应远程分支与之关联 基于当前分支创建并切换分支：git checkout -b [branch] 创建新的远程分支：git push origin [branch] 本地分支与远程分支相互关联：git branch --set-upstream-to=origin/<remote_branch> <local_branch> 为当前工程配置本地的用户名和邮箱： git config --local user.name "user" git config --local user.email "user@example.com" 全局的文件过滤配置：git config --global core.excludesfile [全局的.gitignore文件] 全局的钩子目录配置：git config --global core.hookspath [全局的钩子目录位置.git/hooks] 迁移仓库内所有分支及tag到另一个远程空仓库里 1.本地创建远程分支：git branch -r | grep -v '\->' | while read remote; do git branch --track "${remote#origin/}" "$remote"; done 2.将本地所有分支与远程保持同步: git fetch --all && git pull --all 3.移除之前的origin：git remote remove origin 4.绑定现有的origin：git remote add origin [new_repo.git] 5.提交所有分支代码: git push -u origin --all && git push -u origin --tags 为release分支进行tag操作 ...
