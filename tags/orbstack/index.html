Mac 环境下使用 OrbStack 运行 Docker

概述 在 Mac 上跑 Docker，很多人第一反应还是 Docker Desktop。但如果你的诉求主要是本地容器开发，而不是完整的桌面管理功能，OrbStack 通常是更轻量的一种选择。 结论先说在前面： OrbStack 本质上提供的是一套兼容 Docker CLI 的运行环境 日常使用时，docker、docker compose、镜像构建、端口映射这些命令基本不用改 真正需要关注的是 Docker context、已有 Docker Desktop 的并存关系，以及卷挂载和网络行为 本文只讨论在 Mac 环境下，如何把 OrbStack 当成 Docker 运行时来使用。 为什么在 Mac 上用 OrbStack Mac 不能像 Linux 一样直接原生运行 Docker Engine，通常都要借助虚拟化层。Docker Desktop 和 OrbStack 都是在解决这个问题，只是侧重点不同。 OrbStack 更适合下面这类场景： 主要需求是本地开发、调试和运行容器 希望启动速度更快，常驻资源占用更小 希望继续保留熟悉的 Docker 命令行工作流 经常使用 docker compose 启多个本地服务 如果你的团队脚本、Makefile、CI 辅助命令已经大量使用 docker，那么迁移到 OrbStack 的成本通常比较低，因为多数命令不用改。 安装 OrbStack 在 Mac 上更推荐直接用 Homebrew 安装： brew install --cask orbstack 安装完成后启动 OrbStack，按提示完成初始化即可。 ...
