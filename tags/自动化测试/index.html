把 Android 交给 Agent：Google Android CLI 工程化实战指南

Android CLI 的重点不是“又多了一个 adb 替代品”，而是 Google 正在把 Android 工程能力整理成更适合 AI Agent 调用的命令入口。它把 SDK 管理、项目模板、模拟器、部署、UI 观察、截图定位、官方文档检索、Android skills、Android Studio 语义能力放到统一的 android 命令下。 如果只做手工开发，Android Studio、Gradle 和 adb 已经能覆盖绝大多数工作。但 AI 工程的核心问题不是“人能不能做”，而是“Agent 能不能稳定观察、执行、验证，并留下可复盘证据”。Android CLI 的价值就在这里：它把原本分散在 IDE、终端、设备和文档里的动作，收敛成一组更清晰的命令协议。 Android CLI 可以按下面这几层理解。表格比关系图更适合放在文章开头，读者可以先建立全局印象，再按后文逐段展开。 层级 能力 常用命令 Agent 拿到什么 环境与 SDK 确认 CLI、SDK 和工具链状态 android info、android sdk list/install/update 当前 SDK 路径、可用平台包、工具链是否完整 工程创建与理解 创建模板工程，分析现有项目 android create、android describe、./gradlew tasks module、构建目标、APK 产物路径 设备与运行 管理模拟器并部署应用 android emulator list/start/stop/create、android run 可用设备、安装结果、运行入口 观察与验证 读取 UI 树、截图、定位坐标 android layout、android screen capture/resolve、adb input 页面结构、可点击元素、视觉证据、交互坐标 知识与实践 查询官方资料，安装任务技能 android docs search/fetch、android skills 最新 API 建议、迁移指引、任务级操作规范 IDE 语义能力 复用 Android Studio 索引和预览 android studio analyze-file/find-usages/render-compose-preview 声明位置、引用点、静态分析、Compose Preview 结果 工具边界 1.
