FastAPI快速入门：写第一个REST API

如果你现在学 Python 后端，目标多半不是做传统服务端页面，而是写接口、接数据库、加中间件，给前端或 AI 服务提供 API。这个方向下，FastAPI 比 Flask 更贴近现代开发习惯。 这篇文章不展开讲框架历史，重点只放在实战入门： 写第一个 REST API 理解中间件和数据库接入入口 搞清楚状态码和接口命名规范 先跑起来 安装依赖： pip install fastapi uvicorn 新建 main.py： from fastapi import FastAPI app = FastAPI() @app.get("/") def read_root(): return {"message": "hello"} 运行： uvicorn main:app --reload 启动后访问： http://127.0.0.1:8000/ http://127.0.0.1:8000/docs /docs 是 FastAPI 很实用的一点。接口定义完，文档就能直接看和调试。 写第一个 REST API 先用一个 items 资源做例子： from fastapi import FastAPI, HTTPException, status from pydantic import BaseModel app = FastAPI() # 用内存字典模拟数据库，方便先理解接口结构 items_db = { 1: {"id": 1, "name": "keyboard", "price": 199.0} } class ItemIn(BaseModel): name: str price: float @app.get("/items") def list_items(): # 查询资源列表 return list(items_db.values()) @app.get("/items/{item_id}") def get_item(item_id: int): # 按资源 ID 查询单个对象 item = items_db.get(item_id) if item is None: raise HTTPException(status_code=404, detail="item not found") return item @app.post("/items", status_code=status.HTTP_201_CREATED) def create_item(item: ItemIn): # 创建资源时生成新 ID，并返回 201 new_id = max(items_db.keys(), default=0) + 1 new_item = {"id": new_id, **item.model_dump()} items_db[new_id] = new_item return new_item @app.put("/items/{item_id}") def update_item(item_id: int, item: ItemIn): # PUT 表示整体更新，目标资源不存在时返回 404 if item_id not in items_db: raise HTTPException(status_code=404, detail="item not found") items_db[item_id] = {"id": item_id, **item.model_dump()} return items_db[item_id] @app.delete("/items/{item_id}", status_code=status.HTTP_204_NO_CONTENT) def delete_item(item_id: int): # 删除成功时返回 204，一般不再返回响应体 if item_id not in items_db: raise HTTPException(status_code=404, detail="item not found") del items_db[item_id] 这段代码已经覆盖了最常见的 REST 操作： ...
