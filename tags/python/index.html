Python异步回顾

这篇文章适合已经接触过 Python、但想快速把异步概念捡起来的读者。目标很简单：用最少的内容讲清楚 Python 异步是干什么的、怎么写、哪里最容易出错。 异步是为了解决什么问题 Python 异步主要解决的是 I/O 等待，比如： 同时请求多个接口 同时查多个远程服务 Web 服务在等数据库或网络返回时，不想把线程一直卡住 它不擅长解决 CPU 密集型计算问题。比如大量图片处理、复杂计算，这类任务通常不靠异步提速。 可以先记住一句话： 异步的重点不是“更快执行一段代码”，而是“等待的时候别闲着”。 最核心的三个点 1. async def 定义协程函数 async def fetch_data(): return "ok" 这不是普通函数。调用它时，不会立刻执行，而是先得到一个协程对象。 2. await 表示“先等一下，同时让出执行权” import asyncio async def main(): await asyncio.sleep(1) print("done") asyncio.run(main()) 这里的 await asyncio.sleep(1) 不等于“整个程序卡住 1 秒”，而是“当前任务先暂停，事件循环可以先去执行别的任务”。 3. asyncio.run() 是常见入口 asyncio.run(main()) 可以把它理解成：启动事件循环，把协程跑起来。 最常见的写法区别 很多人刚写异步时，问题不在语法，而在“看起来是异步，其实还是串行”。 串行写法 import asyncio async def work(name, delay): await asyncio.sleep(delay) return f"{name} done" async def main(): r1 = await work("task1", 1) r2 = await work("task2", 2) print(r1, r2) asyncio.run(main()) 这段代码会一个接一个执行，总耗时大约是 3 秒。 并发写法 import asyncio async def work(name, delay): await asyncio.sleep(delay) return f"{name} done" async def main(): r1, r2 = await asyncio.gather( work("task1", 1), work("task2", 2), ) print(r1, r2) asyncio.run(main()) 这里两个任务会一起调度，总耗时大约是 2 秒。 ...
